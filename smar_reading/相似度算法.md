我们在基于向量数据库（如 ChromaDB、Milvus 等）进行检索时，通常会召回与查询向量相似度最高的 Top‑k 个片段。这里的“相似度”本质上是语义层面的相近，即通过 Embedding 模型将文本转换为向量后，比较向量之间的相似程度。

谈到向量相似度的计算方法，很多人会首先想到**余弦相似度**。但实际上，衡量两个向量相似度的方式不止一种，常见的方法还包括：

- **欧式距离（Euclidean Distance）**
- **内积（Dot Product）**
- 以及其他一些度量方式（如曼哈顿距离、汉明距离等，根据场景选择）

其中，在实际应用中最为常用的仍然是**欧式距离**和**余弦相似度**。

## 1. 相似度的配置

不同的 Collection 可以使用不同的相似度算法，在创建 Collection 时通过 `collection_metadata` 参数指定。

支持的距离度量类型：

| 参数值     | 度量方式         | 相似度判断         | 说明                                 |
| :--------- | :--------------- | :----------------- | :----------------------------------- |
| `"l2"`     | 欧氏距离（平方） | **值越小越相似**   | 关注向量的绝对位置，受模长影响       |
| `"cosine"` | 余弦相似度       | **值越大越相似**   | 关注方向夹角，不受模长影响           |
| `"ip"`     | 内积             | **值越大越相似** ⚠️ | 对向量模长敏感，需注意与上两者的差异 |

代码配置：

```python
from langchain_chroma import Chroma


# 方式1：欧氏距离（平方）
chroma_db = Chroma(
    collection_name="test-collection-1",
    embedding_function=DashScopeEmbeddings(),
    persist_directory="./chroma_db",
    collection_metadata={"hnsw:space": "l2"}  # 欧氏距离（平方）
)

# 方式2：通过 collection_metadata 参数指定
chroma_db = Chroma(
    collection_name="test-collection-2",
    embedding_function=DashScopeEmbeddings(),
    persist_directory="./chroma_db",
    collection_metadata={"hnsw:space": "cosine"}  # 余弦相似度
)

# 方式3：内积
chroma_db = Chroma(
    collection_name="test-collection-3",
    embedding_function=DashScopeEmbeddings(),
    persist_directory="./chroma_db",
    collection_metadata={"hnsw:space": "ip"}  # 内积
)
```

**注意事项：**

- ⚠️ 距离度量一旦选定，Collection 创建后**无法修改**
- ⚠️ 不同的 Collection 可以使用不同的度量方式
- ⚠️ 查询时，数据库会自动使用该 Collection 预设的度量方法进行计算

## 2. 欧式距离

欧式距离衡量的是两个向量在向量空间中的**绝对距离**，即两点之间的直线距离。值越小，表示两个向量越相似。

## 3. 余弦相似度

余弦相似度通过计算两个向量之间的夹角余弦值来衡量它们的相似程度，只关注方向，忽略向量的长度（模长）。

## 4. 不同相似度算法的使用场景

| 算法               | 核心关注点            | 适合场景                                                     | 不适合场景                                                   |
| :----------------- | :-------------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| **欧氏距离（L2）** | 向量的绝对位置 + 长度 | • 图像/音频特征检索<br /> • 异常检测 <br />• 需要区分文档长短<br /> • 向量已归一化时 | • 文本语义匹配<br /> • 忽略文档长度的场景<br /> • 高维稀疏向量 |
| **余弦相似度**     | 向量的方向夹角        | • 文本语义相似度<br /> • 用户兴趣画像<br /> • 高维稀疏向量<br /> • RAG 检索增强生成 | • 需要区分信息量大小 <br />• 向量模长有意义时 <br />• 已归一化向量（效果等价 L2） |
| **内积（IP）**     | 方向 + 模长的乘积     | • 推荐系统 CTR 预测 <br />• 注意力机制（Q·K）<br /> • 向量已归一化时 <br />• 需要保留置信度信息 | • 未归一化的文本向量 <br />• 仅需方向比较的场景              |

**快速选择口诀**：

📝 文本语义 → 余弦相似度

🖼️ 图像音频 → 欧氏距离

🤖 推荐匹配 → 内积
